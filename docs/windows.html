Windows RTX 3070 Ti path

Run the Nano pipeline

Goal

Get one path working on Windows: MP4 input, FunASR-Nano ASR, Markdown output.

CUDA is not required for this first pass. Start CPU-only unless runtime is too slow.

1. Install prerequisites

Use PowerShell. Install Go, Git, ffmpeg, and aria2.

winget install GoLang.Go
winget install Git.Git
winget install Gyan.FFmpeg
winget install aria2.aria2

Restart PowerShell after install so PATH updates are visible.

2. Build the binary

If GNU make is available, use the same flow as macOS.

make install
onnx-transcribe --help

If make is not installed, use native Go:

go build -o "$env:USERPROFILE\go\bin\onnx-transcribe.exe" .\cmd\onnx-transcribe
onnx-transcribe --help

3. Install runtime and models

This downloads sherpa-onnx runtime, FunASR-Nano ASR, and Silero VAD into %LOCALAPPDATA%\onnx-transcriber.

onnx-transcribe setup

Because aria2 is installed, setup should show compact download progress with resume support.

4. Check setup

Doctor should find ffmpeg, sherpa binaries, ASR model files, and the VAD model.

onnx-transcribe doctor

5. Copy the smoke MP4

Use a simple path with no spaces or Chinese characters.

mkdir testdata\media
ffmpeg -y -i "$env:USERPROFILE\Downloads\131202 №30431 时空的虚幻 H265-高清-720P.mp4" -t 30 -c copy testdata\media\space-illusion-30s.mp4

6. Run transcription

onnx-transcribe testdata\media\space-illusion-30s.mp4 `
  --threads 8 `
  --out testdata\transcript.smoke.md

VAD is on by default. For the full MP4, use the same command shape with the full file path and write testdata\transcript.full.md.

onnx-transcribe testdata\media\space-illusion-720p-h265.mp4 `
  --threads 8 `
  --out testdata\transcript.full.md

Open the Markdown output and check Mandarin accuracy, paragraph breaks, runtime, and hallucination around silence or music.

If setup fails

  • Run aria2c --version and ffmpeg -version to confirm both are in PATH.
  • Delete only the partial cache file ending in .tmp, then rerun onnx-transcribe setup.
  • For offline install, download archives on another machine and run onnx-transcribe setup --from-dir .\downloads.
