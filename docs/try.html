local smoke guide

Try onnx-transcribe

Use this page as the short path from a fresh checkout to a local MP4 transcription run.

1. Check the repo

Build and test the Go scaffold first.

go test ./...
go run ./cmd/onnx-transcribe --help
go run ./cmd/onnx-transcribe models list

2. Confirm the fixture

The local MP4 is ignored by Git. It should exist here:

testdata/media/131202 №30431 时空的虚幻 H265-高清-720P.mp4

If it is missing, restore it from Downloads:

mkdir -p testdata/media
cp "$HOME/Downloads/131202 №30431 时空的虚幻 H265-高清-720P.mp4" testdata/media/

3. Install assets

Install sherpa-onnx runtime and model assets into the default user data dir.

go run ./cmd/onnx-transcribe setup

For an offline install, put archives in a local directory and use:

go run ./cmd/onnx-transcribe setup --from-dir ./downloads

4. Check setup

Doctor should find ffmpeg, sherpa binaries, and model directories.

go run ./cmd/onnx-transcribe doctor

Current expected pre-setup result: ffmpeg is found, sherpa binaries and model directories are missing.

5. Run the MP4 smoke test

The script writes an ignored Markdown output file.

sh scripts/smoke-mp4.sh

Direct command shape:

go run ./cmd/onnx-transcribe \
  "testdata/media/131202 №30431 时空的虚幻 H265-高清-720P.mp4" \
  --hotwords testdata/hotwords.txt \
  --out testdata/transcript.smoke.md

6. Inspect output

Review readability, hotword hits, punctuation, and hallucination around silence or music.

open testdata/transcript.smoke.md
wc -m testdata/transcript.smoke.md
